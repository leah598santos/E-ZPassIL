# Contributing to E-ZPassIL


Thank you for your interest in contributing to **E-ZPassIL**! We welcome contributions of all sizes — bug reports, documentation, code improvements, tests, and feature ideas.


## Contribution Guidelines


We value clear communication, small focused changes, and respectful collaboration. Please follow these guidelines:


- Follow existing code style and patterns in the repository.
- Keep changes focused to a single purpose per pull request.
- Include or update tests where applicable.
- Write clear commit messages that describe why the change was made.


## How to Contribute


1. Fork the repository to your account.
2. Create a feature branch from `main` (e.g., `feature/add-login`).
3. Make your changes on the feature branch.
4. Run tests and format code before committing.
5. Commit your changes with descriptive messages.
6. Push the branch to your fork and open a pull request against `main`.


## Reporting Issues


If you find a bug or problem:


- Search existing issues to see if it's already reported.
- If not found, open a new issue with a clear title and steps to reproduce, expected vs actual behavior, and environment details.
- Attach screenshots, logs, or sample data where helpful.


## Suggesting Features


Feature requests are welcome! Describe the problem you're trying to solve, why it's important, and a proposed approach if you have one. We’ll discuss feasibility and priority.


## Code of Conduct


By participating in this project, you agree to abide by this Code of Conduct:


- Be respectful and considerate in all interactions.
- Be friendly and welcoming to newcomers.
- Provide constructive criticism and kindly accept feedback.
- Refrain from harassment, discrimination, or personal attacks.


Anyone violating this code may be asked to stop contributing or be blocked from the project.


## Review Process


- Pull requests should include a clear description of changes and link to any related issue.
- Maintainers will review pull requests, leave comments, and request changes if needed.
- Small PRs are preferred; large PRs may be split into smaller ones.
- Once approved and CI checks pass, a maintainer will merge the PR.


Thank you for helping make E-ZPassIL better!
